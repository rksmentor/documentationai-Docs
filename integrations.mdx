## Overview

Enhance your Sourcingmentor experience by integrating with popular third-party tools. You can set up webhooks for real-time notifications, use the API for custom extensions, and handle exports and imports seamlessly. These integrations help automate workflows, sync data with project management tools, and extend functionality without leaving your documentation space.

<Callout kind="tip">
Start with webhooks for simple notifications, then explore API access for deeper customization.
</Callout>

## Webhook Setup

Set up webhooks to receive instant notifications on events like document updates or new pages.

<Steps>
  <Step title="Create Webhook" icon="plus">
    Navigate to Settings > Integrations > Webhooks and click "New Webhook".
  </Step>
  <Step title="Configure Endpoint" icon="link">
    Enter your endpoint URL, such as `https://your-webhook-url.com/webhook`.
  </Step>
  <Step title="Select Events" icon="zap">
    Choose events like `document.updated` or `page.created`.
  </Step>
  <Step title="Test and Save" icon="check-circle">
    Send a test payload and save the configuration.
  </Step>
</Steps>

<ParamField path="event" param-type="string" required="true">
  The event type that triggers the webhook, e.g., `document.updated`.
</ParamField>

<ParamField header="X-Signature" param-type="string" required="false">
  HMAC signature for payload verification using your secret.
</ParamField>

Example webhook payload:

<CodeGroup tabs="JSON">
  ```json
  {
    "event": "document.updated",
    "document_id": "doc_123abc",
    "timestamp": "2024-10-15T10:30:00Z",
    "user_id": "user_456def"
  }
  ```
</CodeGroup>

## API Access for Custom Extensions

Build custom integrations using the Sourcingmentor REST API at `https://api.example.com/v1`.

<Request tabs="cURL,JavaScript" show-lines="true">
  ```bash
  curl -X POST https://api.example.com/v1/documents \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{"title": "New Doc", "content": "Hello world"}'
  ```

  ```javascript
  const response = await fetch('https://api.example.com/v1/documents', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      title: 'New Doc',
      content: 'Hello world'
    })
  });
  ```
</Request>

<Response tabs="200,400">
  ```json
  {
    "id": "doc_123abc",
    "title": "New Doc",
    "status": "success"
  }
  ```

  ```json
  {
    "error": "Invalid API key",
    "status": "400"
  }
  ```
</Response>

<Callout kind="alert">
Always use HTTPS and store `YOUR_API_KEY` securely. Rotate keys regularly.
</Callout>

## Export and Import Options

Export your documentation for backups or migration, and import from other formats.

<Tabs>
  <Tab title="Markdown Export" icon="file-text">
    Export entire spaces or single pages as Markdown files.
    
    ```bash
    curl -H "Authorization: Bearer YOUR_API_KEY" \
      https://api.example.com/v1/spaces/{space_id}/export?format=markdown
    ```
  </Tab>
  <Tab title="PDF Export" icon="file">
    Generate PDF versions for sharing.
    
    ```bash
    curl -H "Authorization: Bearer YOUR_API_KEY" \
      https://api.example.com/v1/pages/{page_id}/export?format=pdf
    ```
  </Tab>
  <Tab title="Import" icon="upload">
    Import Markdown or HTML files via the UI or API.
    
    <Expandable title="Advanced Import Options" default-open="false">
      Use bulk import for large migrations.
    </Expandable>
  </Tab>
</Tabs>
